# GitHub 이슈 관리 가이드라인

이 문서는 프로젝트에서 GitHub 이슈를 효과적으로 관리하기 위한 규칙과 절차를 정의합니다.

## 기본 원칙

### 진행 상황 추적 및 이슈 관리
- **커밋 단위 완료**: 각 커밋은 완전하고 작동하는 기능 증분을 나타내야 합니다
- **GitHub 운영**: 모든 GitHub 관련 작업에는 `gh` CLI를 사용합니다
  - **이슈 관리**: `gh issue` 명령어로 이슈 생성, 업데이트, 코멘트 작성
  - **풀 리퀘스트 관리**: `gh pr` 명령어로 PR 생성 및 관리
  - **리포지토리 운영**: `gh repo`와 `gh api`로 리포지토리 레벨 작업 수행

### 이슈 상태 업데이트
각 중요한 커밋 이후에 해당 GitHub 이슈를 다음과 같이 업데이트해야 합니다:
- `gh issue comment`를 사용하여 완료된 작업을 설명하는 진행 상황 코멘트 추가
- 체크박스를 사용하여 관련 인수 조건을 완료로 표시
- 추적 가능성을 위해 커밋 해시 참조 포함

### 이슈 종료 정책
**변경사항을 커밋하기 전에는 절대 이슈를 종료하지 않습니다**

- 모든 관련 변경사항이 커밋되고 푸시된 후에만 이슈를 종료합니다
- 모든 인수 조건이 충족되고 코드가 커밋된 경우에만 `gh issue close`를 사용합니다
- 작업이 완료되었지만 아직 커밋되지 않은 경우, 완료 코멘트를 추가하되 이슈는 열어둡니다

## 이슈 종료 시 증거 요구사항

GitHub 이슈를 종료할 때는 **반드시** 인수 조건이 충족되었음을 보여주는 증거를 포함해야 합니다.

### 필수 증거 유형
- **로그**: 명령어 출력, 테스트 결과, 배포 로그, 오류 해결 로그
- **기능 데모**: 스크린샷, API 응답, 작동하는 URL, 기능 시연
- **검증 단계**: 사용자가 완료를 재현하고 검증할 수 있는 명확한 지침
- **설정 증거**: 설정값, 환경변수, 배포된 리소스

### 증거 형식 템플릿

```markdown
## ✅ 이슈 완료 증거

### 인수 조건 검증
- [x] 조건 1: [증거/증명]
- [x] 조건 2: [증거/증명]
- [x] 조건 3: [증거/증명]

### 기능 증거
**작동 데모**: [URL/스크린샷/명령어 출력]
**로그**: 
```
[관련 로그 포함]
```

### 검증 지침
**검증 단계**:
1. 테스트할 명령어/URL
2. 예상 결과
3. 재현 방법

**커밋 참조**: [커밋 해시]
```

### 좋은 증거의 예시
- API 엔드포인트가 작동함을 보여주는 `curl` 명령어
- 리소스가 존재함을 보여주는 AWS CLI 명령어
- GitHub Actions 워크플로우 실행 URL 및 상태
- 작동하는 프론트엔드의 스크린샷
- 환경변수 값 (민감하지 않은 정보)
- 테스트 실행 결과

## 실제 적용 예시

### 이슈 생성
```bash
gh issue create --title "기능 제목" --body "상세 설명" --assignee @me
```

### 진행 상황 업데이트
```bash
gh issue comment [이슈번호] --body "✅ 백엔드 API 구현 완료

- JWT 인증 로직 추가
- 사용자 등록 엔드포인트 구현
- 단위 테스트 추가

**커밋**: abc1234"
```

### 이슈 종료
```bash
gh issue close [이슈번호] --comment "✅ 이슈 완료 증거

### 인수 조건 검증
- [x] API 엔드포인트 구현: curl 테스트 성공
- [x] 테스트 커버리지 80% 달성: go test 결과 확인
- [x] 문서 업데이트: README.md 수정 완료

### 기능 증거
**API 테스트 결과**:
```
$ curl -X POST http://localhost:8080/api/users
HTTP/1.1 201 Created
```

**커밋 참조**: abc1234"
```

## 중요 주의사항

1. **커밋 우선**: 항상 코드 변경사항을 먼저 커밋한 후 이슈를 종료합니다
2. **증거 필수**: 모든 이슈 종료 시 구체적인 증거를 제공합니다
3. **추적성 확보**: 커밋 해시와 이슈를 연결하여 추적 가능성을 보장합니다
4. **명확한 검증**: 다른 사람이 결과를 재현할 수 있도록 명확한 지침을 제공합니다

이러한 가이드라인을 따르면 프로젝트의 진행 상황을 체계적으로 추적하고, 모든 작업이 적절히 검증되었음을 보장할 수 있습니다.
